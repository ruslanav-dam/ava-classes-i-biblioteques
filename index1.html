Què és la Modularitat?

Fins ara has fet programes amb UNA SOLA CLASSE... però els programes reals són com ciutats:

Edificis: Molts edificis (classes)
Barris: Organitzats per barris (packages)
Connexions: Connectats entre ells (imports)

El Problema: Programes Monolítics

Programa Monolític

public class ProgramaGegant { // 500 línies de codi... // 50 mètodes diferents... // Tot barrejat! // Impossible de mantenir! }
Problemes:
  • Difícil de llegir
  • No es pot reutilitzar
  • Errors difícils de trobar
  • Impossible treballar en equip

Programa Modular

// CalculsArrays.java public class CalculsArrays { public double calcularMaxim() { ... } public double calcularMinim() { ... } } // RegistreNotes.java public class RegistreNotes { public void inici() { CalculsArrays calc = new CalculsArrays(); calc.calcularMaxim(notes); } }
Avantatges:
  • Fàcil d'entendre
  • Reutilitzable
  • Fàcil de mantenir
  • Treball en equip possible

Exemple Complet: Tot en una classe

public class RegistreNotes { public static void main(String[] args) { RegistreNotes programa = new RegistreNotes(); programa.inici(); } public void inici() { double[] notes = {2.0, 5.5, 7.25, 3.0, 9.5}; double max = calcularMaxim(notes); double min = calcularMinim(notes); System.out.println("Màxima: " + max); } // ⚠️ Aquests mètodes només funcionen aquí! public double calcularMaxim(double[] array) { double max = array[0]; for (int i = 1; i < array.length; i++) { if (max < array[i]) max = array[i]; } return max; } public double calcularMinim(double[] array) { // ... codi similar ... } }

La Solució: Modularitat amb Múltiples Classes

Els 2 Passos Clau

  • INICIALITZAR la classe (crear l'objecte)
    NomClasse identificador = new NomClasse();
  • INVOCAR els seus mètodes
    identificador.nomMetode(parametres);

CalculsArrayReals.java - CODI COMPLET

public class CalculsArrayReals { public double calcularMaxim(double[] array) { double max = array[0]; for (int i = 1; i < array.length; i++) { if (max < array[i]) { max = array[i]; } } return max; } public double calcularMinim(double[] array) { double min = array[0]; for (int i = 1; i < array.length; i++) { if (min > array[i]) { min = array[i]; } } return min; } public double calcularMitjana(double[] array) { double suma = 0; for (int i = 0; i < array.length; i++) { suma += array[i]; } return suma / array.length; } }
✨ Aquesta classe es pot reutilitzar en qualsevol programa que necessiti operar amb arrays!

RegistreNotes.java - CODI COMPLET

public class RegistreNotes { public static void main(String[] args) { RegistreNotes programa = new RegistreNotes(); programa.inici(); } public void inici() { double[] notes = {2.0, 5.5, 7.25, 3.0, 9.5, 8.25}; // Pas 1: INICIALITZEM la classe auxiliar CalculsArrayReals calculador = new CalculsArrayReals(); // Pas 2: INVOQUEM els seus mètodes double max = calculador.calcularMaxim(notes); double min = calculador.calcularMinim(notes); double mitjana = calculador.calcularMitjana(notes); System.out.println("Nota màxima: " + max); System.out.println("Nota mínima: " + min); System.out.println("Mitjana: " + mitjana); } }

Avantatges de la Modularitat

  • Reutilització: Usa CalculsArrayReals en molts programes diferents
  • Organització: Cada classe té una funció clara
  • Manteniment: Si trobes un error, només cal corregir una classe
  • Col·laboració: Diferents programadors poden treballar en diferents classes

Packages: Organitzant Classes

Què és un package?
Un package és com una carpeta que agrupa classes relacionades.
Serveix per organitzar grans projectes amb moltes classes.

Estructura d'un projecte amb packages

src/
utilitats/
arrays/
CalculsArrayReals.java
calcul/
Operacions.java
programa/
RegistreNotes.java
Calculadora.java

Com definir un package

La declaració del package ha de ser la PRIMERA línia del fitxer!
// Fitxer: utilitats/arrays/CalculsArrayReals.java package utilitats.arrays; // ← PRIMERA LÍNIA! public class CalculsArrayReals { // ... el teu codi ... }

Com IMPORTAR classes d'altres packages

  • Importació explícita (RECOMANAT)
    package programa; import utilitats.arrays.CalculsArrayReals; // ← Una classe específica public class RegistreNotes { // Ara pots usar CalculsArrayReals! }
  • Importació amb asterisk (totes les classes)
    import utilitats.arrays.*; // ← TOTES les classes d'aquest package
  • Nom qualificat (sense import)
    // Usar el nom COMPLET cada vegada utilitats.arrays.CalculsArrayReals calc = new utilitats.arrays.CalculsArrayReals();

Convencions de noms per packages

Correcte

package utilitats.arrays; package joc.arena.regles; package empresa.facturacio;

Tot en minúscules
Paraules separades per punts

Incorrecte

package Utilitats.Arrays; package utilitats_arrays; package UTILITATS;

Majúscules
Guions baixos
Tot en majúscules

Exemple Complet: Calculadora Modular

Objectiu

Crear una calculadora amb operacions bàsiques organitzada amb packages.

Estructura del projecte

src/
utilitats/
calcul/
Operacions.java
programa/
Calculadora.java

utilitats/calcul/Operacions.java

package utilitats.calcul; public class Operacions { public int sumar(int a, int b) { return a + b; } public int restar(int a, int b) { return a - b; } public int multiplicar(int a, int b) { return a * b; } public double dividir(int a, int b) { if (b == 0) { System.out.println("Error: divisió per zero!"); return 0; } return (double) a / b; } }

programa/Calculadora.java

package programa; import java.util.Scanner; import utilitats.calcul.Operacions; // ← IMPORTEM! public class Calculadora { public static void main(String[] args) { Calculadora programa = new Calculadora(); programa.inici(); } public void inici() { Scanner sc = new Scanner(System.in); System.out.print("Primer número: "); int num1 = sc.nextInt(); System.out.print("Segon número: "); int num2 = sc.nextInt(); // Inicialitzem la classe d'operacions Operacions ops = new Operacions(); // Usem els seus mètodes System.out.println("Suma: " + ops.sumar(num1, num2)); System.out.println("Resta: " + ops.restar(num1, num2)); System.out.println("Multiplicació: " + ops.multiplicar(num1, num2)); System.out.println("Divisió: " + ops.dividir(num1, num2)); } }

Què hem aconseguit?

  • Separació clara entre lògica d'operacions i programa principal
  • La classe Operacions es pot reutilitzar en altres projectes
  • Codi més fàcil de mantenir i entendre
  • Organització professional amb packages

Exercici 1: Gestió de Text

Objectiu: Crear una aplicació modular per analitzar cadenes de text.

Estructura requerida:

  • utilitats.text.AnalitzadorText - Mètodes d'anàlisi
  • programa.GestorText - Programa principal

Mètodes a implementar en AnalitzadorText:

  • comptarVocals(String text) → retorna int
  • comptarConsonants(String text) → retorna int
  • invertir(String text) → retorna String
  • esPalindrom(String text) → retorna boolean

Exercici 2: Conversió d'Unitats

Objectiu: Crear un conversor d'unitats (temperatura, distància, pes).

Estructura requerida:

  • utilitats.conversions.Temperatura
  • utilitats.conversions.Distancia
  • utilitats.conversions.Pes
  • programa.Conversor - Amb menú d'opcions

Exemple de mètodes:

  • celsiusAFahrenheit(double celsius)
  • kilometresAMilles(double km)
  • kilosALliures(double kg)

Exercici 3: Gestió de Contactes

Objectiu: Crear una agenda de contactes amb operacions bàsiques.

Estructura requerida:

  • utilitats.contactes.GestorContactes
  • programa.Agenda

Funcionalitat:

  • Afegir contacte (nom, telèfon)
  • Buscar contacte per nom
  • Eliminar contacte
  • Mostrar tots els contactes

Pista: Usa arrays paral·lels per emmagatzemar noms i telèfons

Checklist abans d'entregar

  • El projecte està organitzat amb packages correctament
  • Les classes estan en els fitxers i packages correctes
  • Totes les classes tenen la declaració package a la primera línia
  • Els imports són correctes i necessaris
  • El codi compila sense errors
  • El programa funciona correctament
  • El codi està comentat adequadament
