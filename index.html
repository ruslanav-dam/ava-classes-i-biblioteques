Introducció a la Programació Orientada a Objectes

Què és la Programació Orientada a Objectes (POO)?

La Programació Orientada a Objectes és un paradigma de programació que organitza el codi en objectes que contenen tant dades (atributs) com funcionalitat (mètodes).

Analogia del món real

Pensa en un cotxe:

  • Atributs: color, marca, model, velocitat actual, dipòsit de benzina
  • Mètodes: arrencar(), accelerar(), frenar(), girar()

En POO, crearíem una classe Cotxe que defineix aquests atributs i mètodes. Després podríem crear molts objectes cotxe diferents (el meu cotxe, el teu cotxe, etc.)

Per què POO?

Organització

El codi està estructurat de manera lògica, agrupant dades i funcions relacionades

Reutilització

Pots crear una classe i usar-la en múltiples projectes

Mantenibilitat

És més fàcil trobar i corregir errors quan el codi està ben organitzat

Modelatge Real

Representa conceptes del món real de manera natural

Programació Procedural vs POO

Programació Procedural

// Dades i funcions separades double saldoCompte = 1000.0; String titular = "Anna"; void ingressar(double quantitat) { saldoCompte += quantitat; } void retirar(double quantitat) { saldoCompte -= quantitat; }

Les dades i funcions estan separades i desorganitzades

Programació Orientada a Objectes

// Tot agrupat en una classe public class CompteBancari { private double saldo; private String titular; public void ingressar(double quantitat) { saldo += quantitat; } public void retirar(double quantitat) { saldo -= quantitat; } }

Dades i comportament encapsulats junts

Classe vs Objecte

L'analogia de les galetes

Imagina que ets el propietari d'una fàbrica de galetes:

  • La CLASSE és el motlle de les galetes: defineix la forma, la mida, els ingredients
  • Els OBJECTES són les galetes reals que fabriques amb el motlle

Amb un sol motlle (classe) pots fer milers de galetes (objectes). Cada galeta té les mateixes característiques (forma, ingredients) però són galetes individuals i diferents.

Què és una Classe?

Una classe en Java és:
  • Un plànol o plantilla que defineix les característiques i comportaments
  • Conté atributs (variables que representen l'estat)
  • Conté mètodes (funcions que defineixen el comportament)
  • NO ocupa memòria fins que crees objectes

Què és un Objecte?

Un objecte en Java és:
  • Una instància concreta d'una classe
  • Ocupa memòria real quan es crea amb new
  • Té els seus propis valors per als atributs
  • Pot cridar els mètodes definits a la classe

Diferències Clau

Aspecte Classe Objecte
Què és? Plantilla / Plànol / Motlle Instància real creada a partir de la classe
Memòria No ocupa memòria Ocupa memòria quan es crea
Quantitat Una sola definició Molts objectes de la mateixa classe
Paraula clau class new
// UNA classe Persona public class Persona { private String nom; private int edat; public Persona(String nom, int edat) { this.nom = nom; this.edat = edat; } } // MOLTS objectes Persona Persona anna = new Persona("Anna", 25); Persona joan = new Persona("Joan", 30); Persona maria = new Persona("Maria", 22); // Cada objecte té els seus propis valors

Modificadors d'Accés i Encapsulació

Què són els Modificadors d'Accés?

Els modificadors d'accés són paraules clau que controlen qui pot accedir als atributs i mètodes d'una classe.

Analogia de la casa

Pensa en una casa amb diferents habitacions:

  • public: Com el jardí públic - tothom pot entrar
  • private: Com el teu dormitori - només tu hi tens accés

Els dos modificadors principals

Modificador Visible des de... Quan usar-lo
public Qualsevol lloc Mètodes que vols que altres classes usin
private Només dins la pròpia classe Atributs que vols protegir (SEMPRE!)

Getters i Setters

Com que els atributs són privats, necessitem mètodes públics per accedir-hi i modificar-los de manera controlada.

Nomenclatura estàndard:
  • Getter: get + NomAtribut() - per obtenir el valor
  • Setter: set + NomAtribut(valor) - per establir el valor
public class Persona { // Atributs PRIVATS private String nom; private int edat; // GETTER per nom public String getNom() { return nom; } // SETTER per nom amb validació public void setNom(String nom) { if (nom != null && !nom.isEmpty()) { this.nom = nom; } } }

Regla d'or de l'encapsulació:

  • Atributs → SEMPRE private
  • Mètodes d'ús públic → public
  • Usar getters/setters per accedir als atributs

Constructors i la Paraula Clau THIS

Què és un Constructor?

Un constructor és un mètode especial que s'executa automàticament quan crees un objecte amb new.

La seva funció: Inicialitzar l'estat de l'objecte amb valors inicials.

Per què necessitem Constructors?

SENSE Constructor (Malament)

public class Persona { private String nom; private int edat; } // Crear objecte sense inicialitzar Persona p = new Persona(); // nom = null, edat = 0 // Atributs sense valors vàlids!

L'objecte es crea amb valors per defecte (null, 0) que poden causar errors

AMB Constructor (Correcte)

public class Persona { private String nom; private int edat; // Constructor que inicialitza public Persona(String nom, int edat) { this.nom = nom; this.edat = edat; } } // Crear objecte AMB valors Persona p = new Persona("Anna", 25); // nom = "Anna", edat = 25 // Objecte correctament inicialitzat!

L'objecte es crea amb valors vàlids des del principi

Característiques dels Constructors

  • Mateix nom que la classe
    Si la classe es diu Persona, el constructor es diu Persona (exactament igual)
  • NO té tipus de retorn
    Ni void, ni int, ni res. Simplement no en té.
  • S'executa automàticament
    Quan fas new NomClasse(), el constructor s'executa sol
  • Pot rebre paràmetres
    Per inicialitzar l'objecte amb valors específics que tu decideixes

La Paraula Clau THIS - Explicació Profunda

Què és THIS?

this és una referència a l'objecte actual, l'objecte que s'està creant en aquest moment.

Serveix per distingir entre els atributs de la classe i els paràmetres del constructor quan tenen el mateix nom.

Per què necessitem THIS?

Imagina aquesta situació:

public class Persona { private String nom; ← Atribut de la classe public Persona(String nom) { ← Paràmetre del constructor // Aquí hi ha DOS "nom" diferents! // 1. this.nom = l'atribut de la classe // 2. nom = el paràmetre del constructor } }

Sense THIS - ERROR!

public Persona(String nom, int edat) { nom = nom; // ❌ Assigna el paràmetre a si mateix! edat = edat; // ❌ No modifica l'atribut! } // Resultat: Persona p = new Persona("Anna", 25); // Els atributs queden null i 0 // Els valors "Anna" i 25 es perden!

Problema: Java no sap si "nom" és l'atribut o el paràmetre, així que usa el paràmetre.

Amb THIS - CORRECTE!

public Persona(String nom, int edat) { this.nom = nom; // ✅ atribut = paràmetre this.edat = edat; // ✅ atribut = paràmetre } // Resultat: Persona p = new Persona("Anna", 25); // Els atributs són "Anna" i 25 // Funciona perfectament!

Solució: this.nom indica clarament que és l'atribut de la classe.

THIS en Detall - Visualització

Quan es crea un objecte:

Persona anna = new Persona("Anna", 25); // Java fa això internament: // 1. Crea espai en memòria per a un objecte Persona // 2. "this" apunta a aquest nou objecte // 3. Executa el constructor: // this.nom = "Anna" → L'atribut nom d'aquest objecte = "Anna" // this.edat = 25 → L'atribut edat d'aquest objecte = 25 // 4. Retorna la referència a l'objecte creat // 5. "anna" ara fa referència a aquest objecte

Exemple Complet amb THIS Explicat

public class CompteBancari { // Atributs de la classe private String titular; private double saldo; // Constructor public CompteBancari(String titular, double saldo) { // this.titular fa referència a l'atribut de la classe // titular (sense this) fa referència al paràmetre this.titular = titular; this.saldo = saldo; } // També podem usar this en altres mètodes public void mostrarInfo() { // Aquí this és opcional (no hi ha ambigüitat) // però és bona pràctica usar-lo per claredat System.out.println("Titular: " + this.titular); System.out.println("Saldo: " + this.saldo); } }

Constructors amb Validació

Els constructors són el lloc ideal per validar que les dades siguin correctes abans d'inicialitzar l'objecte.

public class Estudiant { private String nom; private int edat; private double nota; public Estudiant(String nom, int edat, double nota) { // Validar nom if (nom != null && !nom.trim().isEmpty()) { this.nom = nom; } else { this.nom = "Desconegut"; System.out.println("⚠️ Nom invàlid, s'usa 'Desconegut'"); } // Validar edat (entre 16 i 99) if (edat >= 16 && edat <= 99) { this.edat = edat; } else { this.edat = 18; System.out.println("⚠️ Edat invàlida, s'usa 18"); } // Validar nota (entre 0 i 10) if (nota >= 0.0 && nota <= 10.0) { this.nota = nota; } else { this.nota = 0.0; System.out.println("⚠️ Nota invàlida, s'usa 0.0"); } } }

Avantatges de validar en el constructor:

  • Seguretat: Els objectes sempre es creen amb dades vàlides
  • Consistència: No pots tenir objectes en estat invàlid
  • Detecció primerenca d'errors: Els problemes es detecten immediatament

Múltiples Constructors (Sobrecàrrega)

Pots tenir diversos constructors amb diferents paràmetres. Java decideix quin usar segons els arguments que li passis.

public class Producte { private String nom; private double preu; // Constructor 1: sense paràmetres public Producte() { this.nom = "Producte genèric"; this.preu = 0.0; } // Constructor 2: només amb nom public Producte(String nom) { this.nom = nom; this.preu = 0.0; } // Constructor 3: nom i preu public Producte(String nom, double preu) { this.nom = nom; this.preu = preu; } } // Ús: Producte p1 = new Producte(); // Constructor 1 Producte p2 = new Producte("Portàtil"); // Constructor 2 Producte p3 = new Producte("Ratolí", 25.99); // Constructor 3

Errors Comuns amb Constructors

Els 3 errors més freqüents:

// ❌ ERROR 1: Posar tipus de retorn public void Persona() { // Això NO és un constructor, és un mètode normal! } // ❌ ERROR 2: Nom diferent de la classe public inicialitzar() { // Això és un mètode normal, no un constructor } // ❌ ERROR 3: No usar 'this' public Persona(String nom) { nom = nom; // Assigna el paràmetre a si mateix! No funciona! }

Resum: Per què THIS és IMPRESCINDIBLE

  • Claredat: Deixa clar que estàs modificant l'atribut de la classe
  • Evita errors: Sense this, els atributs no s'inicialitzen
  • Bones pràctiques: És l'estàndard en Java professional
  • Mantenibilitat: El codi és més fàcil d'entendre i mantenir

Regla simple: En els constructors, SEMPRE usa this.atribut = parametre

Packages: Organitzant el Codi

Què és un Package?

Un package és com una carpeta que agrupa classes relacionades.

Funció: Organitzar grans projectes amb moltes classes i evitar conflictes de noms.

Declarar un Package

Regla important: La declaració del package ha de ser la PRIMERA línia del fitxer
// Fitxer: agenda/Contacte.java package agenda; ← PRIMERA LÍNIA! public class Contacte { private String nom; // ... }

Importar Classes

// Importar una classe específica import agenda.Contacte; public class Principal { public static void main(String[] args) { Contacte c = new Contacte("Anna", "666777888"); } }

"Sí" : "No")); } }
