El Problema: Programes Monolítics

Programa Monolític

public class ProgramaGegant { // 500 línies de codi... // 50 mètodes diferents... // Tot barrejat! // Impossible de mantenir! }
Problemes:
  • Difícil de llegir
  • No es pot reutilitzar
  • Errors difícils de trobar
  • Impossible treballar en equip

Programa Modular

// CalculsArrays.java public class CalculsArrays { public double calcularMaxim() { ... } public double calcularMinim() { ... } } // RegistreNotes.java public class RegistreNotes { public void inici() { CalculsArrays calc = new CalculsArrays(); calc.calcularMaxim(notes); } }
Avantatges:
  • Fàcil d'entendre
  • Reutilitzable
  • Fàcil de mantenir
  • Treball en equip possible

public class CalculsArrayReals { public double calcularMaxim(double[] array) { double max = array[0]; for (int i = 1; i < array.length; i++) { if (max < array[i]) { max = array[i]; } } return max; } public double calcularMinim(double[] array) { double min = array[0]; for (int i = 1; i < array.length; i++) { if (min > array[i]) { min = array[i]; } } return min; } public double calcularMitjana(double[] array) { double suma = 0; for (int i = 0; i < array.length; i++) { suma += array[i]; } return suma / array.length; } }

Packages: Organitzant Classes

Què és un package?
Un package és com una carpeta que agrupa classes relacionades.
Serveix per organitzar grans projectes amb moltes classes.

Estructura d'un projecte amb packages

src/
utilitats/
arrays/
CalculsArrayReals.java
calcul/
Operacions.java

RegistreNotes.java
Calculadora.java

Com definir un package

La declaració del package ha de ser la PRIMERA línia del fitxer!
// Fitxer: utilitats/arrays/CalculsArrayReals.java package utilitats.arrays; // ← PRIMERA LÍNIA! public class CalculsArrayReals { // ... el teu codi ... }

Com IMPORTAR classes d'altres packages

  • Importació explícita (RECOMANAT)
    package programa; import utilitats.arrays.CalculsArrayReals; // ← Una classe específica public class RegistreNotes { // Ara pots usar CalculsArrayReals! }
  • Importació amb asterisk (totes les classes)
    import utilitats.arrays.*; // ← TOTES les classes d'aquest package
  • Nom qualificat (sense import)
    // Usar el nom COMPLET cada vegada utilitats.arrays.CalculsArrayReals calc = new utilitats.arrays.CalculsArrayReals();

Convencions de noms per packages

Correcte

package utilitats.arrays; package joc.arena.regles; package empresa.facturacio;

Constructors: Inicialitzant Objectes

Què és un constructor?
Un constructor és un mètode especial que s'executa automàticament quan crees un objecte amb new.
Serveix per inicialitzar l'estat de l'objecte amb valors inicials.

Característiques dels Constructors

  • Mateix nom que la classe: Si la classe es diu Persona, el constructor es diu Persona
  • NO té tipus de retorn: Ni void, ni int, ni res
  • S'executa automàticament: Quan fas new NomClasse()
  • Pot rebre paràmetres: Per inicialitzar l'objecte amb valors específics

Constructor per Defecte (sense paràmetres)

Sense Constructor

public class Persona { private String nom; private int edat; // Sense constructor explícit // Java crea un constructor buit automàticament }

Java crea un constructor buit automàticament, però no inicialitza res

Amb Constructor

public class Persona { private String nom; private int edat; // Constructor que inicialitza public Persona() { nom = "Desconegut"; edat = 0; } }

El constructor estableix valors inicials per defecte

Constructor amb Paràmetres

El constructor més útil és el que rep paràmetres per inicialitzar l'objecte amb valors específics:

public class Persona { private String nom; private int edat; // Constructor amb paràmetres public Persona(String nom, int edat) { this.nom = nom; this.edat = edat; } public void mostrarInfo() { System.out.println("Nom: " + nom + ", Edat: " + edat); } }
Importància de 'this':
this.nom = nom; significa: "l'atribut nom d'aquest objecte = el paràmetre nom"
this fa referència a l'objecte actual i evita confusions entre atributs i paràmetres amb el mateix nom.

Usar el Constructor

public class ProgramaPrincipal { public static void main(String[] args) { // Crear objecte passant valors al constructor Persona p1 = new Persona("Anna", 25); Persona p2 = new Persona("Joan", 30); p1.mostrarInfo(); // Nom: Anna, Edat: 25 p2.mostrarInfo(); // Nom: Joan, Edat: 30 } }

Múltiples Constructors (Sobrecàrrega)

Pots tenir diversos constructors amb diferents paràmetres:

public class Persona { private String nom; private int edat; // Constructor 1: sense paràmetres public Persona() { nom = "Desconegut"; edat = 0; } // Constructor 2: només amb nom public Persona(String nom) { this.nom = nom; edat = 0; } // Constructor 3: amb nom i edat public Persona(String nom, int edat) { this.nom = nom; this.edat = edat; } }
Ús:
Persona p1 = new Persona(); → Usa el constructor 1
Persona p2 = new Persona("Maria"); → Usa el constructor 2
Persona p3 = new Persona("Pere", 40); → Usa el constructor 3

Errors Comuns amb Constructors

Incorrecte

// ERROR 1: Tipus de retorn public void Persona() { // Això NO és un constructor! } // ERROR 2: Nom diferent public inicialitzar() { // Això és un mètode normal } // ERROR 3: No usar 'this' public Persona(String nom) { nom = nom; // Assigna el paràmetre a si mateix! }

Correcte

// CORRECTE 1: Sense tipus de retorn public Persona() { // Aquest és un constructor vàlid } // CORRECTE 2: Mateix nom que la classe public Persona(String n) { nom = n; } // CORRECTE 3: Usar 'this' public Persona(String nom) { this.nom = nom; // Clar i sense ambigüitat }

Resum: Quan usar Constructors

  • Inicialització d'atributs
    Quan la teva classe té atributs que necessiten valors inicials
  • Validació de dades
    Per comprovar que els valors inicials són correctes (per exemple, edat no pot ser negativa)
  • Flexibilitat
    Crear múltiples constructors dona opcions a qui usa la teva classe
  • Codi més net
    Inicialitzar en el constructor és més clar que fer-ho després amb setters

Conceptes clau

  • Constructor: Mètode especial per inicialitzar objectes
  • Mateix nom: Ha de tenir exactament el mateix nom que la classe
  • Sense return: Mai té tipus de retorn, ni void
  • this: Fa referència a l'objecte actual
  • Sobrecàrrega: Pots tenir múltiples constructors amb diferents paràmetres

package utilitats.calcul; public class Operacions { public int sumar(int a, int b) { return a + b; } public int restar(int a, int b) { return a - b; } public int multiplicar(int a, int b) { return a * b; } public double dividir(int a, int b) { if (b == 0) { System.out.println("Error: divisió per zero!"); return 0; } return (double) a / b; } }

programa/Calculadora.java

package programa; import java.util.Scanner; import utilitats.calcul.Operacions; // ← IMPORTEM! public class Calculadora { public static void main(String[] args) { Calculadora programa = new Calculadora(); programa.inici(); } public void inici() { Scanner sc = new Scanner(System.in); System.out.print("Primer número: "); int num1 = sc.nextInt(); System.out.print("Segon número: "); int num2 = sc.nextInt(); // Inicialitzem la classe d'operacions Operacions ops = new Operacions(); // Usem els seus mètodes System.out.println("Suma: " + ops.sumar(num1, num2)); System.out.println("Resta: " + ops.restar(num1, num2)); System.out.println("Multiplicació: " + ops.multiplicar(num1, num2)); System.out.println("Divisió: " + ops.dividir(num1, num2)); } }

Què hem aconseguit?

  • Separació clara entre lògica d'operacions i programa principal
  • La classe Operacions es pot reutilitzar en altres projectes
  • Codi més fàcil de mantenir i entendre
  • Organització professional amb packages
